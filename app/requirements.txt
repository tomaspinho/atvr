# Requirements for pyatv 0.16.1 with --no-deps.
#
# pyatv 0.16.1 declares cryptography>=44.0.1 and chacha20poly1305-reuseable
#>=0.13.2 (which needs cryptography>=43.0.0). Neither has an Android wheel on
# Chaquopy beyond 42.0.8. However, the cryptography APIs pyatv actually uses
# (ed25519, x25519, HKDF, Cipher, serialization) are stable since
# cryptography 2.x, so 42.0.8 works at runtime. chacha20poly1305-reuseable
# 0.13.2 is a one-liner re-export of cryptography's ChaCha20Poly1305.
#
# pyatv 0.16.1 is the latest version that still allows pydantic 1.x
# (>=1.10.10), which is pure Python and has no pydantic-core dependency.
# pyatv 0.17.0+ requires pydantic>=2.0.0, which pulls in pydantic-core
# (a Rust native extension with no Android wheel on Chaquopy).
#
# We use --no-deps globally (set in build.gradle.kts) and list every package
# in the full dependency tree explicitly, so pip's resolver doesn't reject
# the "incompatible" version pins. No local/cross-compiled wheels needed.
